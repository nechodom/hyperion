## 📋 Roadmapa pro zbytek tvého seznamu

Setříděno podle effort + value. Každá položka má **odhad času pro
seniora**, **závislosti**, a **co potřebuju od tebe** (designové
rozhodnutí, schéma, nebo content).

### 🔥 Production-blocking / nejvyšší priorita

#### `backups-delete-transfer` · ~1-2 dny
- **Co**: Smazat zálohu, přesunout zálohu na jinou node.
- **Co chybí**: `backup_delete` + `backup_transfer` RPC, UI buttons, audit.
- **Otázky**: 
  - Transfer = stream přes mTLS mezi nodes? Nebo přes shared remote backup target?
  - Hard-delete vs soft (přesun do tombstone)?

#### `wp-install-at-create` · ~2-3 dny
- **Co**: Při vytvoření hostingu vyplnit doménu + WP creds + HTTPS, vše jedním klikem.
- **Co chybí**: Nový create flow co řadí provisioning → DNS check → ACME → WP install.
- **DNS check**: rezolvovat A/AAAA, porovnat s public IP node, varovat preflight.
- **Otázky**: Tvoje preferred admin email pattern? Default pluginy?

#### `per-web-acme-email-override` · ~3-4 hodiny
- **Co**: DB field `hostings.acme_contact_email` (nullable), edit form, fallback na cfg.acme.contact_email.
- **Závislosti**: SQL migrace 12.

### 🟧 Vysoká hodnota, středně náročné

#### `monitoring-per-site` · ~2-3 dny
- **Co**: On/off per hosting, email + Slack alerty, threshold.
- **Otázky**:
  - Co monitorovat? (HTTP 200 check, response time, disk usage, mail delivery?)
  - Alerty: hned, nebo po N selháních za řadou?
  - Webhook formát? (Slack incoming webhook, generic JSON?)

#### `service-health-check` · ~1-2 dny
- **Co**: Dashboard widget "all services healthy", per-service status + auto-restart, notifikace na výpadek.
- **Závislosti**: Email config už máš.

#### `reverse-proxy-hosting-type` · ~2-3 dny
- **Co**: Nový typ hostingu = reverse proxy na URL, ne PHP/static. nginx `proxy_pass` template.
- **Otázky**:
  - Custom headers, websocket, gzip, cache?
  - Per-proxy SSL upstream nebo plain HTTP?
  - Auth (Basic, headers forwarding)?

#### `file-browser-in-admin` · ~3-4 dny
- **Co**: Browse + edit souborů hostingu z UI.
- **Bezpečnost**: path-traversal protection, owner check, max file size, MIME whitelist for inline edit.
- **Otázky**:
  - Edit mode (read-only / per-role)?
  - Permission model — kdo smí číst/editovat?

#### `multi-admin-users` · ~2 dny
- **Co**: Více admin uživatelů s vlastními creds, invite flow, password reset.
- **Závislosti**: Email config už máš (pro invites).
- **Otázky**: Audit log podle usera (už máme `actor_label` plumbing).

#### `roles-and-per-web-assignments` · ~3-4 dny  
- **Co**: Roles (super-admin / admin / operator / user), per-hosting RBAC (kdo vidí/edituje který web).
- **Závislosti**: multi-admin-users hotový.
- **Otázky**:
  - Konkrétní role + permissions matrix?
  - Hierarchie nebo flat?

#### `2fa-totp` · ~2 dny
- **Co**: TOTP enrollment, backup codes, login flow, recovery.
- **Závislosti**: žádné, lze samostatně.

#### `2fa-enforcement-per-user` · ~půl dne
- **Co**: Toggle "tenhle user musí mít 2FA", uvedeno v UI.
- **Závislosti**: 2fa-totp hotové.

### 🟨 Středně priorita

#### `dashboard-redesign` · ~2-3 dny
- **Co**: "Hezčí, využij celou plochu". 
- **Otázky**: 
  - Co ti na současné nesedí? (Layout? Density? Cards? Charts?)
  - Mockup nebo reference? (CloudPanel/Plesk/RunCloud screenshoty?)
- **Tip**: Začnu draftem layoutu s ~4-6 widget bloky, pošlu screenshot k revizi.

#### `app-wide-settings-page` · ~2 dny
- **Co**: `/settings` — všechno co je teď v `agent.toml` a `web.toml` v UI: ACME email, default backup target, retention, Slack, Email SMTP, atd.
- **Otázky**: Co je per-node vs cluster-wide?

#### `server-node-settings` · ~2 dny
- **Co**: Per-node settings (PHP defaults, max hostings, label, public IP).
- **Závislosti**: app-wide-settings hotový.

#### `backup-target-config` · ~2 dny  
- **Co**: UI pro nastavení FTP/SFTP/S3 backup destinace.
- **Závislosti**: existuje už infra pro `backup_remote` v agent.toml — jen UI + secrets storage.

#### `wp-with-profiles` · ~3-4 dny
- **Co**: Pre-prepared WP install profil s pluginy + theme (operator nahraje, použije při install).
- **Závislosti**: Storage pro plugin/theme zipy, validace.

#### `ftp-own-tab-multi-users` · ~2 dny
- **Co**: FTP do vlastní tab v hosting detail, podpora víc FTP uživatelů per hosting.
- **Závislosti**: PAM-based vsftpd — víc users = víc Linux users → komplikace.

#### `postfix-email-test` · ~1 den
- **Co**: Test odeslání emailu z UI, log delivery, případně SPF/DKIM check.
- **Závislosti**: SMTP config už máš. Postfix as relay je samostatný setup.

#### `pretty-creds-display` · ~půl dne
- **Co**: "Ready to copy" panel po vytvoření hostingu se všemi credentials (DB pass, FTP pass, WP admin), click-to-copy.

### 🟦 Bezpečnostní průřez

To rozházet napříč všemi feature commity (ne samostatný úkol):
- Validace všech inputs (už máš `hyperion-validate`)
- `#![forbid(unsafe_code)]` všude
- Audit log pro každý state-change (už máš pattern)
- Secrets v `hyperion-core::secrets::SecretsStore`
- mTLS pro multi-node communication (až bude)
- Rate limiting na login (zatím chybí)
- CSRF tokens (už máš)
