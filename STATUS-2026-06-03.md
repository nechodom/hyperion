## 📋 Co jsem **nestihl** (zámerně skipnuto, design needs you)

Tyhle položky vyžadují tvé designové rozhodnutí, nebo jsou samy o sobě velké features (1-3 dny solid práce každá):

| Feature | Proč nestihl | Co potřebuju vědět |
|---|---|---|
| **Multi-admin users + 2FA** | Security-critical, invite flow + password reset + 2FA recovery musí být designované společně. ~3-5 dní práce. | TOTP nebo WebAuthn? Recovery codes 10 nebo 12? Enforcement per-user vs per-role? |
| **Roles / per-web RBAC** | Vyžaduje schema design (role table, user_role, user_hosting_access), migration je nereversibilní. ~3-4 dny. | Konkrétně jaké role + permissions matrix? |
| **Reverse proxy hosting type** | Nginx template variant, websocket pass, auth headers, upstream SSL. ~2-3 dny. | Jen URL nebo i auth/headers/websocket? Per-proxy cert? |
| **File browser** | Bezpečnostně netriviální — path traversal, MIME whitelist, max size, edit mode. ~3-4 dny. | Read-only nebo i edit? Kdo smí (operator vs site-owner)? |
| **Monitoring per-site + alerts** | Co měřit? Threshold logika? Alert dedup window? Slack/email channel? ~3 dny. | HTTP 200, response time, disk? Hned alert nebo po N selháních? |
| **WP install at create flow** | Refactor create form + flow + WP installer reuse. ~2 dny. | Default plugins/theme? Auto-issue HTTPS? |
| **Backup transfer between nodes** | mTLS plumbing nebo shared S3? Velký design choice. ~2-3 dny. | mTLS p2p, nebo přes shared backup target? |
| **Multi-FTP-users per hosting** | vsftpd PAM-based = nový Linux user per FTP user. Nasypaný interface. ~2 dny. | Limit count? Per-FTP-user chroot path? |
| **WP profiles s pluginy** | Zip upload + storage + deploy on install. ~2-3 dny. | Operator-uploaded zips, nebo URL z Git? |
| **App settings editace (write)** | toml_edit + backup + hot-reload nebo restart. ~1-2 dny. | Restart agent po edit, nebo SIGHUP? |

Tyhle všechny máš na svém wishlistu — když mi řekneš priority a designové rozhodnutí, šlapu dál. Kód, který je hotový, je solid foundation.
