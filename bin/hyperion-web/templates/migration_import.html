Transfer file location

On the source node, open the hosting, choose Move or copy → Export a transfer file, then create the file. The result page shows a base URL and a token — paste both here.

What happens next

  1. This node downloads manifest.json and archive.tar.gz from the source over the signed URL.
  2. The archive's BLAKE3 digest is checked against the manifest, so a corrupted transfer is refused.
  3. A fresh hosting is provisioned with the source's domain, aliases, PHP version, and database engine.
  4. The archive (htdocs plus the database dump) is restored.
  5. You're taken to the new hosting's detail page.
  6. Point DNS at this node, then click Issue certificate once it resolves. Private keys never travel between nodes.
  7. On the source, run hctl hosting delete <domain> once you've confirmed the imported site works.
