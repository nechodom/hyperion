Bundle URL

On the source node, open the hosting detail → Migration tab → click Export migration bundle. The result page prints a base URL and a token — paste both here.

What happens next

  1. This node downloads manifest.json + archive.tar.gz from the source via the signed URL.
  2. BLAKE3 of the archive is verified against the manifest — a corrupted transfer is refused.
  3. A fresh hosting is provisioned with the source's domain, aliases, PHP version, DB engine.
  4. The archive (htdocs + DB dump) is restored.
  5. You'll be redirected to the new hosting's detail page.
  6. Update DNS to point at this node; click Issue Cert when LE-ready (private keys never travel across nodes).
  7. On the source: hctl hosting delete <domain> once you've verified the imported site works.
