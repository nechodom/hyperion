HTTPS & HSTS

HSTS tells browsers to refuse plain HTTP for this domain. Once a browser sees the header it remembers for the full max-age, so values above one day are effectively permanent — to undo, you must serve max-age 0 for that long.

Maintenance mode

Use during deploys or database restores. Browsers see a generic "we'll be back" page; Let's Encrypt renewals continue because the /.well-known/acme-challenge/ path bypasses the 503.

FastCGI page cache

Bypassed automatically for logged-in WordPress / PHP-session visitors (checks wordpress_logged_in_* and PHPSESSID cookies). The cache zone is per-hosting.

Security · WAF-lite

Conservative nginx rules: deny direct access to wp-config.php, xmlrpc.php, dumps and backups; block PHP execution under wp-content/uploads and cache; and 403 obvious probe query-strings and scanner user-agents.

{# Per-hosting actions must dispatch to the node where this hosting actually lives — without this, suspend/resume/set-limits/backup/ etc. on a worker hosting silently no-op on the master. Each per-hosting form gets a hidden input target_node before submit; the value comes from the detail handler (find_hosting_ anywhere). When target_node is empty (master), the input is still appended with value="" which the server reads as "local" — preserves the existing default. This is a JS shim instead of editing every form's HTML so new forms don't have to remember to add the field. Worst case (browser without JS): forms go to master, which is correct for master-hosted sites and surfaces a clear NotFound error for worker-hosted ones. #} {% endblock %}
