Backup targets

Off-site destinations for scheduled hosting backups. Generic S3-compatible (Wasabi, Backblaze B2, Minio, AWS). Operator-supplied credentials + an age public key for client-side encryption. The FTP/FTPS/SFTP single-target push (live today) + local retention live on Settings → Backups.
← Settings
How it runs. Every time a hosting backup completes (manual, from the hosting detail page) the agent also pushes it to each enabled S3 target here — archive + DB dump + manifest. With an age recipient set, each object is encrypted client-side and streamed (age → aws, no temp file), so the off-site copy needs no extra local disk. After uploading, the remote is pruned to the newest “keep daily” backups per hosting. Requires apt install awscli (and age for encryption) on each node. Note: the weekly/monthly tiers below currently map to “keep newest = daily”; full grandfather-father-son tiering is a later refinement.
