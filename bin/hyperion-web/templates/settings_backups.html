Backup targets

Off-site destinations for scheduled hosting backups, on any S3-compatible storage (Wasabi, Backblaze B2, MinIO, AWS). Each target holds your credentials and an age public key for client-side encryption. The single-target FTP/FTPS/SFTP push and local retention live on Settings → Backups.
← Back to settings
How it runs. Whenever a hosting backup completes (manually, from the hosting detail page), the agent also pushes it to every enabled S3 target here — archive, database dump, and manifest. With an age recipient set, each object is encrypted client-side and streamed (age → aws, no temp file), so the off-site copy needs no extra local disk. After upload, the remote is pruned to the newest “keep daily” backups per hosting. Each node needs the AWS CLI (apt install awscli), plus age for encryption. Note: the weekly and monthly tiers below currently map to “keep newest = daily”. Full grandfather-father-son tiering is a later refinement.
