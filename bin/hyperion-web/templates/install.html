How node enrollment works

  1. Generate a labeled token above. Plaintext shown once — copy immediately; only its hash is persisted server-side.
  2. Paste the printed curl … | sudo bash … on the new VPS.
  3. The install script provisions deps, builds hyperion-agent, saves the token + master URL into /etc/hyperion/agent.toml, and starts the service.
  4. Full mTLS enrollment (sub-project 1.5 in the design specs) consumes the token and rolls the new node into the cluster. Until that lands the node runs single-node and the operator can manage it via its own socket.
