How enrollment works

  1. Generate a labeled token above. Plaintext shown once — copy immediately; only its hash is persisted server-side.
  2. Paste the printed curl … | sudo bash … on the new VPS.
  3. Install script provisions deps, builds hyperion-agent, saves the token + master URL into /etc/hyperion/agent.toml, and starts the service.
  4. Full mTLS enrollment (sub-project 1.5) consumes the token and rolls the new node into the cluster. Until that lands the node runs single-node.
