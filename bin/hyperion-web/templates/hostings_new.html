Hyperion downloads the latest WordPress core, seeds the database, creates the admin user, and (if a profile is picked) installs every plugin and theme it lists. You'll see a live progress card on the hosting page while it runs — usually 10–60 seconds depending on plugin count.

Required by the selected profile. WordPress is installed automatically so the profile's plugins and themes have somewhere to land. To skip the WP install entirely, change the profile back to None / custom in step 1.

Pick a non-obvious admin username — admin is the first guess every brute-forcer tries. WordPress is installed at the site root after provisioning and you'll see the admin credentials in the credentials panel.

What happens next

  1. Linux user created with chroot-style home.
  2. Directory tree laid out under /home/{user}/{domain}.
  3. PHP-FPM pool rendered and the service reloaded.
  4. Database + DB user provisioned, password returned once.
  5. TLS cert issued; nginx vhost goes live.

Any step that fails triggers LIFO rollback — the system never half-creates a hosting.
