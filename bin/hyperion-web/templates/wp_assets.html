How this works

  • Upload a plugin or theme ZIP. Hyperion validates the ZIP's magic bytes, deduplicates by SHA-256, and stores the file under /var/lib/hyperion/wp-assets/<id>/<sanitized-filename>.
  • Open a hosting profile's edit page (Profiles → Edit) and add lines like @asset:7! to the Plugins or Themes field.
  • A trailing ! on the line also activates the asset after install.
  • When the profile is applied to a WordPress-enabled hosting, the agent runs wp plugin install <path> --force (or wp theme install) as that hosting's system user. --force replaces an existing install, which is useful when you upload a newer ZIP under the same internal slug.
  • Profiles can mix uploaded assets with WordPress.org slugs (akismet, yoast-seo) in the same list.
