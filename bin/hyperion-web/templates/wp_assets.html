How this works

  • Upload a plugin or theme ZIP. Hyperion validates the ZIP magic bytes, dedupes on SHA-256, and stores the file under /var/lib/hyperion/wp-assets/<id>/<sanitized-filename>.
  • Open a hosting profile's edit page (Profiles → Edit) and add lines like @asset:7! to the Plugins or Themes textarea.
  • Trailing ! on the line means "also activate after install".
  • When the profile is applied to a WordPress-enabled hosting, the agent shells out to wp plugin install <path> --force (or wp theme install) under that hosting's system user. --force replaces an existing install — handy when you upload a newer ZIP under the same internal slug.
  • Profiles can mix uploaded assets with WordPress.org slugs (akismet, yoast-seo) in the same list.
